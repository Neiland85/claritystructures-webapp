# Decisión de no tocar código antes de presentación

## Decisión

No tocar código antes de presentación salvo bug bloqueante real.

## Por qué

Porque ya hay dos activos demostrables:

- web en producción para demo guiada;
- Vault con demo técnica reproducible.

Seguir tocando código ahora puede introducir ruido, bugs y dispersión.

## Qué toca ahora

- preparar narrativa;
- enseñar demo;
- validar interés;
- hablar con perfiles cualificados;
- medir respuesta;
- decidir inversión.

## Qué no toca ahora

- más refactors;
- más polish visual;
- más endpoints;
- más features sin cliente;
- mezclar otros proyectos;
- convertir la demo en producto masivo sin validación.
