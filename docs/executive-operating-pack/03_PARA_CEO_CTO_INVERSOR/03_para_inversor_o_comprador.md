# Explicación para inversor o comprador

## Qué se compra

Se compra un activo técnico-comercial en fase de validación avanzada.

No es solo repositorio. Es:

- producto;
- narrativa;
- documentación;
- demostración;
- mercado potencial;
- metodología;
- oportunidad de verticalización.

## Por qué puede interesar

Porque combina varias áreas con demanda creciente:

- legaltech;
- evidence governance;
- cybersecurity;
- compliance;
- digital forensics;
- auditability;
- data governance;
- trust infrastructure.

## Qué reduce riesgo de compra

- web operativa;
- demo técnica reproducible;
- artefacto ZIP verificable;
- tests pasando;
- límites comunicados;
- documentación ejecutiva;
- no se sobrepromete certificación.

## Qué aumenta valoración

- pilotos pagados;
- usuarios cualificados;
- tráfico segmentado;
- partnerships;
- validación por despachos;
- revisión técnica externa;
- integración con procesos reales;
- métricas de conversión.

## Qué pregunta debe hacerse un comprador

No: “¿cuántas líneas de código tiene?”

Sí:

- ¿qué problema caro resuelve?
- ¿quién lo compra?
- ¿cuánto cuesta adquirir cliente?
- ¿cuánto falta para producto vendible?
- ¿qué barrera técnica/documental tiene?
- ¿qué equipo puede escalarlo?
